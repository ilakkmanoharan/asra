--------------------------------------------------
1. ARC-AGI-3 Runner
--------------------------------------------------

Create an environment runner that can:

- Initialize an ARC-AGI-3 game/environment.
- Reset the environment.
- Receive a frame from the environment.
- Parse the frame.
- Send an action.
- Receive the next frame.
- Track terminal states:
  - NOT_FINISHED
  - WIN
  - GAME_OVER

The runner should expose:

class ArcAGI3Runner:
    def reset(self) -> Frame
    def step(self, action: str) -> StepResult
    def get_available_actions(self) -> list[str]
    def run_episode(self, agent, max_steps: int) -> EpisodeResult

Supported actions:

RESET
ACTION1
ACTION2
ACTION3
ACTION4
ACTION5
ACTION6
ACTION7

Do not assume action semantics. The agent must infer action effects through experimentation.

2. Frame Parser
--------------------------------------------------

Create a frame parser that converts raw ARC-AGI-3 JSON frames into a normalized internal format.

Expected normalized frame:

{
  "game_id": "...",
  "level_id": "...",
  "step_index": 0,
  "grid": [[0,1,2], [3,4,5]],
  "height": 3,
  "width": 3,
  "status": "NOT_FINISHED",
  "metadata": {
    "raw_frame_keys": [],
    "timestamp": "...",
    "source": "arc-agi-3"
  }
}

Parser must validate:

- Grid exists.
- Grid is rectangular.
- Grid max size is <= 64x64.
- Cell values are integers from 0 to 15.
- Status is valid.
- Coordinate system assumes top-left origin: row = y, column = x.

--------------------------------------------------
8. Dead-End Detector
--------------------------------------------------

Implement simple dead-end detection.

A state is a possible dead-end if:

- All tested actions return to same state.
- All actions produce no meaningful grid change.
- The agent repeatedly cycles among already visited states.
- The environment reaches GAME_OVER.
- No action increases novelty after N attempts.

Dead-end output:

{
  "state_hash": "...",
  "dead_end_score": 0.85,
  "reasons": [
    "all_actions_tested",
    "no_grid_change",
    "cycle_detected"
  ],
  "recommended_action": "RESET"
}

Use this initially as a heuristic, not a final truth.

--------------------------------------------------
9. Simple Exploration Policy
--------------------------------------------------

Build baseline policy:

Policy name:
simple_exploration

Behavior:

- Prefer untested actions in current state.
- Prefer actions that create new state hashes.
- Avoid actions known to cause GAME_OVER.
- Avoid repeated cycles.
- Occasionally retry actions if stuck.
- Use RESET if dead-end score is high.
- Stop episode on WIN, GAME_OVER, or max_steps.

Policy scoring:

score(action) =
  novelty_bonus
  + grid_change_bonus
  + terminal_win_bonus
  - cycle_penalty
  - game_over_penalty
  - no_change_penalty

Return selected action plus reasoning:

{
  "selected_action": "ACTION3",
  "reason": "untested_action_with_high_novelty_potential",
  "score": 1.75
}
